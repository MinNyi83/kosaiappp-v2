Deployment & Infrastructure Console

Setup guides for native Docker Synology NAS mounts and routing utilizing free, zero-config trycloudflare.com tunnels.

1. Quick Tunnel Setup (trycloudflare.com)

Route traffic on-the-fly to your local server without a Cloudflare account. Ideal for field technician testing.

Start trycloudflare Tunnel locally (bash)
# Run a transient cloudflared docker instance on your host docker run --rm --name temp-cloudflare-tunnel --network host cloudflare/cloudflared:latest tunnel --url http://localhost:8080 # Check container output logs to retrieve the generated *.trycloudflare.com url: # Example: +------------------------------------------------------------+ # Example: | Your quick tunnel has been created! | # Example: | URL: https://random-generated-sub.trycloudflare.com | # Example: +------------------------------------------------------------+

2. Synology Volume Direct Docker Mounts

Mount your Synology NAS shared folder directly inside your docker-compose.yml using native NFS/CIFS drivers.

docker-compose.yml NFS & CIFS volume options
# --- OPTION A: NATIVE NFS VOLUME DECLARATION --- volumes: synology-nfs-uploads: driver: local driver_opts: type: "nfs" o: "addr=192.168.1.100,rw,nolock,hard,nfsvers=4" device: ":/volume1/service-uploads" # --- OPTION B: NATIVE CIFS/SMB VOLUME DECLARATION --- volumes: synology-smb-uploads: driver: local driver_opts: type: "cifs" o: "username=nas_username,password=nas_password,rw,vers=3.0,domain=WORKGROUP" device: "//192.168.1.100/service-uploads"

3. trycloudflare Unified Docker Compose Stack

A single file mapping the volume mount to the container and running cloudflared as a transient quick tunnel.

docker-compose.yml (trycloudflare quick tunnel version)
version: '3.8' services: # The Service Management Dashboard Application service-app: build: context: . dockerfile: Dockerfile container_name: awesomemyanmar-app ports: - "8080:80" volumes: # Code volume - ./:/usr/share/nginx/html:ro # Synology mounted folder for uploads - /var/www/awesomemyanmar/uploads:/usr/share/nginx/html/uploads:rw restart: unless-stopped # Cloudflare transient quick tunnel (Zero Login required) cloudflare-tunnel: image: cloudflare/cloudflared:latest container_name: cf-tunnel-try # Tunnel to the service-app container on port 80 dynamically command: tunnel --no-autoupdate --url http://service-app:80 restart: unless-stopped depends_on: - service-app
