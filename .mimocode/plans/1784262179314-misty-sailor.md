## Steps

### Step 1: Clean local D1 state
Delete any stale `.wrangler/state/v3/d1/` files to ensure a fresh start.

### Step 2: Initialize local D1 database
Run `schema.sql` first (creates core 11 tables with `PRAGMA foreign_keys = OFF`), then run the incremental migrations in order:

```bash
npx wrangler d1 execute DB --local --file=db/migrations/schema.sql
npx wrangler d1 execute DB --local --file=db/migrations/roles_config_sync.sql
npx wrangler d1 execute DB --local --file=db/migrations/inventory_sync.sql
npx wrangler d1 execute DB --local --file=db/migrations/create_credits_table.sql
npx wrangler d1 execute DB --local --file=db/migrations/mock_data.sql
npx wrangler d1 execute DB --local --file=seed_admin_new.sql
```

**Key**: Each SQL file sets `PRAGMA foreign_keys = OFF` at the start, so FK ordering issues during migration are suppressed. The final `PRAGMA foreign_keys = ON` enables enforcement at runtime.

### Step 3: Verify
- Start dev server briefly to confirm it boots: `npm run dev`
- Confirm the local DB is created under `.wrangler/state/v3/d1/`
